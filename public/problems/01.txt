#TITLE 01 - Appointment Tracking
#SUMMARY Task 01: Patient Queuing System using Lists and Functions

## Problem Overview
A Hospital Management System requires a front-desk software solution to help queue patients for appointments efficiently. Your task is to develop a foundational script that manages patient data using global lists and modular functions.

---

### Core Requirements
To complete this task, your Python script must implement the following:

1.  **Data Structure**: Establish a **Global List** that can store patient records (Name, Age, and Gender).
2.  **Modular Function**: Create a function named `add_patient` that accepts three parameters: `name`, `age`, and `gender`. This function should append the patient's data to the global list.
3.  **Outcome Verification**: 
    *   Initialize the system.
    *   Add a male patient named "Ali", age 17.
    *   Add a female patient named "Sara", age 28.
    *   Print all stored patient data to the console.

---

### Expected Output Structure
When executed, your program should display the patient data clearly, for example:
```text
Patient Added: Ali (17, M)
Patient Added: Sara (28, F)
```

---

### Implementation Guide
> [!TIP]
> **Variable Scope**: Remember to handle global variables carefully within your functions. In Python, you can directly append to a global list without the `global` keyword, but descriptive naming helps maintainability.

> [!IMPORTANT]
> A single `.py` file is required to complete this task.
